2017-10-26

基础命令

ls命令

ls -l
显示详细的文件列表
ls -
按照上一次修改时间排序文件

cp命令

cp -p fileA fileB
从A文件的内容拷贝到B文件
cp fileA direB
将A文件复制到B目录

file

file A.png 输出文件信息
a.txt: ASCII text

less

less a.txt 快速查看文件

top

显示系统状况
PID — 进程id
USER — 进程所有者
PR — 进程优先级
NI — nice值。负值表示高优先级，正值表示低优先级
VIRT — 进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
RES — 进程使用的、未被换出的物理内存大小，单位kb。RES=CODE+DATA
SHR — 共享内存大小，单位kb
S — 进程状态。D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
%CPU — 上次更新到现在的CPU时间占用百分比
%MEM — 进程使用的物理内存百分比
TIME+ — 进程使用的CPU时间总计，单位1/100秒
COMMAND — 进程名称（命令名/命令行）

tar

tar命令可以为linux的文件和目录创建档案。利用tar，可以为某一特定文件创建档案（备份文件）
首先要弄清两个概念：打包和压缩。打包是指将一大堆文件或目录变成一个总的文件；压缩则是将一个大的文件通过一些压缩算法变成一个小文件。
必要参数有如下：
-A 新增压缩文件到已存在的压缩
-B 设置区块大小
-c 建立新的压缩文件
-d 记录文件的差别
-r 添加文件到已经压缩的文件
-u 添加改变了和现有的文件到已经存在的压缩文件
-x 从压缩的文件中提取文件
-t 显示压缩文件的内容
-z 支持gzip解压文件
-j 支持bzip2解压文件
-Z 支持compress解压文件
-v 显示操作过程
-l 文件系统边界设置
-k 保留原有文件不覆盖
-m 保留文件不被覆盖
-W 确认压缩文件的正确性
可选参数如下：
-b 设置区块数目
-C 切换到指定目录
-f 指定压缩文件
–help 显示帮助信息
–version 显示版本信息
tar -cvf log.tar log2012.log 仅打包，不压缩！
tar -zcvf log.tar.gz log2012.log 打包后，以 gzip 压缩
tar -zcvf log.tar.bz2 log2012.log 打包后，以 bzip2 压缩

ps(process status)列举系统运行进程的状态

a 显示所有进程
-a 显示同一终端下的所有程序
-A 显示所有进程
c 显示进程的真实名称
-N 反向选择
-e 等于“-A”
e 显示环境变量
f 显示程序间的关系
-H 显示树状结构
r 显示当前终端的进程
T 显示当前终端的所有程序
u 指定用户的所有进程
-au 显示较详细的资讯
-aux 显示所有包含其他使用者的行程
-C<命令> 列出指定命令的状况
–lines<行数> 每页显示的行数
–width<字符数> 每页显示的字符数
–help 显示帮助信息
–version 显示版本显示

2017入秋

2017-10-14


国庆过后的第一个周末，秋凉袭来，听着张国荣的《春夏秋冬》心情异常平静。
看了会数据结构的书，过了两个钟，终究还是停了下来。觉得这灰蒙蒙的天气适合做点其他事。
我在狭小的屋子走来走去，若有所思，然后坐下来泡了一壶红茶。翻着过去空间写的日志，如此文艺，以至于之前的朋友对我转行写代码觉得不可思议。
转行写代码之后，很少有时间和心情思索一些文艺的事情。偶尔听到过去感动过的歌，才把我带入短暂的思索。
人变化太快，取决于思想的变化。
看着过去在微博和空间发表的心情记录。才发现自己变得那么多，有点惊讶，也有点失落。
因为当时心里一直坚持的想法，现在变得无所谓了。
像是辜负了、背叛了曾经的自己。但是，现在的心里好像也在鄙视着之前的自己，幼稚。
红茶在秋天凉得很快，张国荣的歌也在一首一首播着，这片时光带着秋天的气息，在歌中流淌着，往后的日子听着张国荣应该可以更方便地回忆这段生活。
只是，这样的心情、这样的日子真的越来越少了。
像今天的周末无所事事的日子并不多，不能希冀着夏日过后的第一个秋凉来惊醒自己。
忙碌的生活，更需要抽离自己，站在远处，观望自己，思索此时此时的生活。
忙碌的一生也并不值得庆贺。
而我，希望通过一些文字，在未来仍然很好的回忆起现在的生活。

前端面试题收录

2017-10-12
1、cookie和session

首先，http是无状态的协议。出现session和cookie都是为了解决无状态协议带来的问题。通过session和cookie，后端就知道客户端是谁。服务端为了记录每个用户的行为，创建了特定session保存在服务端。而cookie则是给服务端识别身份用的，每次进行http请求的时候，客户端会带上一个特定的sessionid或者说某个服务端第一次在客户端设置的sessionid。总的来说，session在服务端保存用户信息，依赖于保存在客户端的一个sessionid、而cookie在客户端保存了用户的信息一般是包含某个sessionid，在http传递时候给服务端用。

2、解释let暂时性死区和块级作用域

let 和 const都有暂时性死区，即当进入它的作用域，它不能被访问（获取或设置）直到执行到达声明。

let块级作用域会在包围他的地方形成作用域，在作用域内，声明的变量不会覆盖作用域外的变量，并且只能声明一个同样的变量。

3、工厂模式、策略模式、装饰者模式

策略模式

策略模式是指对一系列的算法定义，并将每一个算法封装起来，而且使它们还可以相互替换。策略模式让算法独立于使用它的客户而独立变化。

6、 typeof和instanceof

相同点：JavaScript 中 typeof 和 instanceof 常用来判断一个变量是否为空，或者是什么类型的。
typeof的定义和用法：返回值是一个字符串，用来说明变量的数据类型。
Instanceof定义和用法：instanceof 用于判断一个变量是否属于某个对象的实例。

(1)、typeof 一般只能返回如下几个结果：number,boolean,string,function,object,undefined。

(2)、typeof 来获取一个变量是否存在，如 if(typeof a!=”undefined”){alert(“ok”)}，而不要去使用 if(a) 因为如果 a 不存在（未声明）则会出错。

(3)、对于 Array,Null 等特殊对象使用 typeof 一律返回 object，这正是 typeof 的局限性。

BFC-块级格式化上下文

2016-07-23
块级格式化上下文 是web页面可视html渲染的一部分。它是块盒子的布局发生及浮动体彼此交互的区域。

简单来说，BFC能创建一个内部与外界互不影响的盒子。在实际项目中，盒子外边距折叠、包容盒子坍塌之类的情况可以用BFC原理解决。

块级格式化上下文由以下之一创建：
  • 根元素或其它包含它的元素
  • 浮动 (元素的 float 不是 none)
  • 绝对定位的元素 (元素具有 position 为 absolute 或 fixed)
  • 内联块 inline-blocks (元素具有 display: inline-block)
  • 表格单元格 (元素具有 display: table-cell，HTML表格单元格默认属性)
  • 表格标题 (元素具有 display: table-caption, HTML表格标题默认属性)
  • 块元素 元素具有overflow 值不是 visible
  • display: flow-root
